How Forest works

The tree

Everything you build lives in the tree on the left. Each item is a node. Nodes can contain other nodes, forming pages and their pieces — a page node might contain section nodes, which might contain further nodes, and so on. The very top node, All, is the root of everything you own.

  • Click a node to open it in the editor on the right.
  • Click the arrow next to a node to expand or collapse its children.
  • Double-click a node's name to rename it.
  • Drag and drop a node onto another to move it — drop near the top of a node to move inside it as the first child, or lower to place it after it as a sibling.

The toolbar

  • New — creates a new child node under whichever node is currently selected.
  • Duplicate — makes a copy of the selected node (and everything inside it), placed right after the original.
  • Delete — removes the selected node. A node must be empty (no children) before it can be deleted.
  • Save — saves your changes to the currently open node.
  • Preview / Publish — only shown for page nodes (names ending in .html); see below.
  • Search — searches the name and content of your nodes.

Editing a node

Every node has a Name and a Template — the HTML that defines how it renders. Inside the template, you can reference any field you've added using double curly braces, e.g. {% raw %}{{title}}{% endraw %}.

Two variables are always available, without needing a field:

  • {% raw %}{{name}}{% endraw %} — the node's own name.
  • {% raw %}{{html}}{% endraw %} — renders all of the node's children in order.

Use + Field to add a new piece of content to the node — text, a rich text block, an image, a link, and more. Once added, a field shows up as its own input below the template; double-click its label to edit its type or options again.

Preview & Publish

Any node whose name ends in .html is a page. For pages, two extra buttons appear in the toolbar:

  • Preview opens a new tab showing what the page (and everything nested inside it) currently renders to — without saving anything to disk.
  • Publish renders the page and writes it out as a static HTML file, mirroring your tree's folder structure. This is what actually makes a page live.

Publishing always walks up to the nearest ancestor page (or All) first, so publishing any node inside a page republishes that whole page.

Search

Type into the search box and press Enter to find nodes by name or content. Click a result to jump straight to it in the editor.

Sharing & ownership

You only see and edit your own nodes. If you're an admin, you can also see nodes created by the admin account. User accounts and roles are managed from the Users page.
